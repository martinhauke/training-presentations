# E2E-Testing with Cypress

<span style="margin-bottom: 10px; text-align: left; display: block; width: 100%">
  <strong>Pros:</strong>
</span>

<ul style="font-size: 40px">
  <li>full e2e (you need an actual running application somewhere), so even API calls etc. are being covered</li>
  <li>screenshots + videos of the test results</li>
</ul>

<span style="margin: 20px 0 10px; text-align: left; display: block; width: 100%">
  <strong>Cons:</strong>
</span>

<ul style="font-size: 40px">

  <li>slow</li>
  <li>it might be necessary to mock external services, depending on the environment the tests are running in</li>

</ul>

<Notes>

Recommendation:

- use react-testing-library for integration tests
- use cypress for e2e-testing
- depending on your app you might not need both

**Show examples in redux-hands-on repo**

</Notes>
