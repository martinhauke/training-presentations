# Test Driven Development (TDD)

<ul>

<Appear>

<li>Approach where tests are written before the actual implementation of a unit/feature</li>
<li>Red-green-cycle (test -> implementation -> refactor)</li>
<li>Helps formulate the requirements of a feature</li>
<li>Reduces the amount of manual testing necessary -> fast feedback loop</li>
<li>Removes the necessity to write tests afterwards</li>

</Appear>

</ul>
