# Behavior Driven Development (BDD)

<ul>

<Appear>

  <li>Done with tools like <strong>cucumber</strong></li>
  <li>Typically done in a TDD fashion</li>
  <li>Tests are written in plaintext using <i>Gherkin</i>-syntax (or <i>Behat</i> etc.)</li>
  <li>Gherkin is just a set of grammar rules</li>
  <li>Tests are called <i>Scenarios</i> and consist of <i>steps</i></li>
  <li>A step is usable Gherkin-text which wraps actual code</li>
  <li>Steps have to be defined programatically by developers</li>
  <li>Steps can contain test setup/fixtures as well as assertions</li>

</Appear>

</ul>
