Language Primer

aka ECMAScript

Transpilation & Runtime

In modern JS/TS projects we usually transpile our source code to the actual runtime code. To do so we use tools like Babel or the Typescript compiler. Benefits:

  • Use modern language features not yet implemented by all browsers
  • Cross browser compatibility
  • Typescript as typed super set of JS

var vs let & const

  • Differences to var:
    • block scoped
    • variables are inaccessible before initialization (hoisting works differently)
    • const can't be re-assigned and must be initialized during declaration (BUT: referenced objects could still be mutated)
    • prefer const over let
    • (almost) never use var

Values & References

  • Primitives (booleans, numbers & strings) are assigned by value
console.log(42 === 42); // => true
  • Reference types (arrays, objects and functions) are assigned by reference
console.log({} === {}); // => false (someReference === otherReference)

Arrow Functions

  • Shorter syntax for the lazy
  • Think lambdas from other languages
  • Has no own this or arguments[] and other gotchas
  • Binds this and arguments from outer scope
  • Not hoisted like function
